# [GitHub] affaan-m/ECC

[github.com/affaan-m/ECC](https://github.com/affaan-m/ECC) (~212k★) · curated skill pack **with** an opt-in learning loop.

The key counter-example to "optimizers can't be popular." Mostly a curated pack, but ships a real loop: **Instincts (Continuous Learning v2)** — hooks auto-extract patterns into confidence-scored "instincts"; `/evolve` clusters them into skills (work → extract → score → promote). Plus `/skill-create` from git history, SessionStart/End memory hooks. (v1 used a Stop-hook to observe; v2 moved observation to PreToolUse/PostToolUse for 100% capture.)

## Skill / experience accumulation flow (verified 2026-06-23)

The full loop, two-tier (cheap always-on **instincts** → promoted **skills**), with a clean split between the **use/forward** path and the **improve/reverse** path:

1. **Observe (capture).** `observe.sh` on **PreToolUse / PostToolUse** hooks + a background **Haiku observer** analysing in parallel. Hook-driven = **100% deterministic capture** (the stated reason for moving off v1's skill/Stop-hook observation, which "fires ~50–80% of the time"). Project detected via `~/.claude/homunculus/projects.json`.
2. **Distil → stage (optimistic write, no gate).** Patterns become **atomic instincts** written to a **staging store outside `.claude/`**: `CLV2_HOMUNCULUS_DIR` (default `~/.local/share/ecc-homunculus`); per-project at `~/.claude/homunculus/projects/<id>/instincts/personal/`. Each instinct: **Action + Evidence** (no Examples); **confidence** = Haiku-stamped per-window frequency band (0.5/0.7/0.85), **write-once**; **domain tag** (code-style/testing/git/debugging/workflow/file-patterns); **scope** project|global.
3. **Curate — *documented, not implemented*.** Code audit (2026-06-24): **no rolling curate exists.** confidence is **write-once**; the decay-if-contradicted spec lives only in the orphan `agents/observer.md` (zero code references) — see autopsy. Only real post-generation move is **manual, deterministic scope-promotion**. No structural conflict scan, and no working scalar curate either.
4. **Use / inject (forward path).** **SessionStart only** (`session-start.js`; **UserPromptSubmit is NOT registered**), **char-capped** (`ECC_SESSION_START_MAX_CHARS` default 8000; `…_CONTEXT=off` disables). Pure JS, no model: filter **confidence ≥ 0.7** → sort conf-desc, project>global, **id-alphabetical** → **top 6**. No per-prompt relevance retrieval; the "adherence evidence" step 3 supposedly consumes is **never measured**. (Resolves the prior open question: top-6-by-confidence with a 0.7 floor, then 8000-char cap.)
5. **Evolve → promote (gated entry to the live skill layer).** `/evolve` clusters high-confidence instincts into **skills/commands** at `$ECC_AGENT_DATA_HOME/skills/learned/` (default root `~/.claude`). `/skill-create` also mints skills from git history. `/instinct-import` · `/instinct-export` move instincts in/out.
6. **Skill-layer maintenance (periodic, manual — the "A" route).** `/skill-stocktake` audits skills (overlap / staleness / usage → **Keep/Improve/Update/Retire/Merge**, destructive needs confirm; Quick re-checks only changed via `results.json`+mtime). `rules-distill` lifts cross-skill principles (in 2+ skills) into **rules**, human-approved. Philosophy: **"deterministic collection + LLM judgment"** and **"What, not How"**.
7. **Skill use.** Promoted skills then ride **native description-relevance triggering (~50–80%)** — which is *why* must-always-apply behaviour is deliberately kept as **hook-injected instincts (100%)**, not skills.

**Two reliability tiers (the load-bearing design choice):** instinct→hook = **100%** deterministic; skill→description = **50–80%** model-judged. ECC routes high-reliability rules to hooks and reserves the skill path for "occasionally relevant."

```
PreTool/PostTool hooks ─► instincts (staging: ~/.local/share/ecc-homunculus, conf+domain+scope)
   observe                    │  (NO decay/curate in code — write-once) · manual project→global
                              ├─► USE: SessionStart-only inject (conf≥0.7, top-6, ≤8000 chars; pure JS)
                              └─► /evolve ─► skills/learned/ ─► description-trigger (~50–80%)
                                            skill-stocktake · rules-distill (periodic, human-approved)
```

## Code-level autopsy (verified 2026-06-24 — full source read of ECC@main)

Read the actual scripts, not docs (`observe.sh`, `agents/observer-loop.sh`, `scripts/hooks/session-start.js`, `continuous-learning-v2/scripts/instinct-cli.py`, `agents/observer.md`, `hooks/hooks.json`). **The running system is far thinner than the docs claim.** Whole loop = **capture → one Haiku `--print` call → inject**; everything past generation (rank / curate / evict) is absent, unscientific, or manual.

- **Capture** (`observe.sh`, Pre+PostToolUse `*`): each tool call → 2 JSONL records (tool_start/tool_complete) — tool + input + output (truncated 5000, **secret-scrubbed**), session, project. **Not the user prompt** → "user correction" patterns are *inferred from the tool trace* (re-edits/reverts), never read.
- **Observer** = `observer-loop.sh`, a **`nohup` shell daemon**, **off by default** (`observer.enabled=false`), lazy-started + lock-guarded by observe.sh, one per project, **idle-exits 30 min**. On SIGUSR1 (every 20 obs) / 60s cooldown / ANALYZING guard → `tail -n 500` of observations.jsonl → **one** call: `… claude --model haiku --print --allowedTools "Read,Write" -p "<inline prompt>"`. **走套餐 confirmed**: Haiku via the `claude` CLI reuses Claude Code auth (subscription if logged in), not a separate API key.
- **Generation = one inline prompt, no algorithm.** The whole "learning" is that prompt: find 3+-occurrence patterns (user-corrections / error-resolutions / repeated-workflows / tool-preferences), Write `<id>.md`, `confidence by frequency 3-5=0.5/6-10=0.7/11+=0.85`, "if similar exists update", "global if universal", "never include code snippets". **No code defines pattern / same / duplicate / conflict** — all Haiku gestalt over the 500-line tail; **non-deterministic, non-reproducible**. "Frequency" is **per-window** (observations archived after each run; **>500 between runs = unanalyzed blind spot**).
- **Injection** (`session-start.js`): **pure JS, no model, SessionStart only** (UserPromptSubmit not registered). filter **conf ≥ 0.7** → sort conf-desc, project>global, **id-alphabetical** → **top 6** → `Active instincts:\n- [scope conf%] <Action line>` as additionalContext, bundle capped 8000. Coarse 2-bucket sort (0.5 is below the floor; injected = only 0.7/0.85) + alphabetical tie-break ⇒ "confidence ranking" carries **no merit signal**.
- **No post-creation evolution.** Exhaustive grep (all .js/.py/.sh): **zero code reads-then-rewrites a confidence value.** confidence is **write-once** (and even mis-stamped — the shipped example is 0.7 for "5 instances", which the prompt's own rule calls 0.5). decay/boost (`+0.05`/confirm, `-0.1`/contradict, `-0.02`/week staleness) lives ONLY in `agents/observer.md`, which **nothing loads** (grep: 0 refs) and which **disagrees** with the runtime prompt. Only non-generation moves: deterministic manual scope-promotion (`promote`, 2+ projects & avg≥0.8) and Haiku overwrite-on-regenerate.
- **No eviction of active instincts.** Three 30-day TTLs exist, **none touch active instincts**: `prune` → `instincts/pending/` only (auto-gen lands in `personal/`); observe.sh purges `observations-*.jsonl`; session-start.js purges `*-session.tmp`. No decay → no death-by-threshold; the 0.7 inject floor only **dormants** low-conf instincts (kept on disk forever). orphan observer.md specs a `-0.02/week` staleness decay — **designed, not implemented**. **Net: active instincts only grow — never expire, never retire.** (`skill-evolution/health.js` computes success-rate-7d/30d + failure_trend, but it's **read-only reporting** for human skill-stocktake — skill layer, not instincts; mutates nothing.)

**Corrections to this card's 2026-06-23 doc-based claims** (ECC's own docs oversell the loop): anatomy is **Action+Evidence** (no Examples); inject is **SessionStart only** (not +UserPromptSubmit); "rolling curate / decay" is **not implemented** (orphan-spec); confidence is a per-window prompt instruction, not a computed lifetime score; "adherence evidence" is **never measured**.

**What autoharness can actually borrow (narrowed by this audit):** only **hook capture of tool I/O · one-shot Haiku generation · hook injection**. ECC entirely lacks — so autoharness must build — (1) scientific ranking/priority from real usage signal, (2) relevance-aware retrieval (vs SessionStart top-6), (3) real eviction (structural dedup/conflict/subsume **+ staleness retire**), (4) a true **non-regressing rolling curate**. This is the wedge, now code-confirmed.

**Mapping to autoharness:** the **reverse/improve** half (distil · curate · evolve · stocktake · rules-distill) is our territory (Intake + Manage); the **forward/use** half (inject/trigger) is host-native — we shape artifacts, the host applies. Borrow: "deterministic collection + LLM judgment", incremental re-scan (changed-only), the Keep/Improve/Update/Retire/Merge verdict set, staging-outside-`.claude` + gated promotion, and the **vehicle-by-reliability** rule (skill vs hook vs always-loaded). Don't inherit: ECC has **no structural dedup/conflict gate** (overlap only, periodic, manual) — that gated, conflict-aware admission is our wedge.

**Relevance to autoharness:** the category wins when packaged as "install-and-it-works content pack + an optional learning loop hidden inside," not as a research framework. ECC owns the **distribution** corner; [SkillOpt-Sleep](microsoft-skillopt-sleep.md) owns engineering, [RHO](../papers/rho.md) owns validation method — nobody has all three.
