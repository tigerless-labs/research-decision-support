# [博客] 文档腐烂 — 最大但竞争涌入的痛点

行业口径：80% 知识库过期、仅 19.1% 支持人员称文档"很准确"；arXiv context-rot 研究：
82 仓库 23% 检出失效代码引用（人工确认 64% 为真）。信任螺旋："Don't trust the wiki"
→ 工程师绕过文档直接 grep 源码。

2026 趋势：被动存储 → 主动维护（工具监听产品变化自动同步）、docs 从工作系统派生、
"知识作为工作副产品被捕获"。竞争者已在涌入。

**与 loom 的关系**：doc 管理的思维模式（one fact one home、docs 只装 why、链接校验、
索引实时同步）本仓库的工作规则已在实践——可提炼为检查器型 skill，但此赛道拥挤度最高。

[FastDoc 2026](https://www.fastdoc.io/blog/state-of-software-documentation-2026)、
[context rot (arXiv)](https://arxiv.org/pdf/2606.09090)。
