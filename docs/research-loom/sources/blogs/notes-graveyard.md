# [博客] "笔记坟场" — 2026 年 second brain 圈的共识痛点

多篇 2026 文章同一诊断（Substack/Medium/工具评测）。

核心逻辑：笔记系统"善进坟出"——capture 强、retrieval 无（"funnels with no drain"）；
Zettelkasten/PARA 把维护负担全压在人身上，两周热情后归档成坟；"capture without
retrieval is hoarding"。2026 主流回应是 **AI 维护 vault**：agent 扫描全库、分类
active/stale/superseded/promotable（已有 Obsidian × Claude Code skill 这么做，tagline
"a vault that doesn't review itself is a filing cabinet with extra steps"）。

**与 loom 的关系**：验证 idea 管理痛点为真且赛道已被注意；loom 的差异位不是"AI 帮你
回顾"，而是想法自带**生命周期与证据义务**（status 枚举 + 必须链出处 + 未被使用可见）。
