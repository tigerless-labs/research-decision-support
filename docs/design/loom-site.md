# loom 多页工作台

一个 workspace 渲染成四个自包含 HTML 页面，覆盖"读 → 标 → 判"主循环；面向正在读材料的
作者本人，溯源图退为面向读者的第四页。通用于任何"来源→设计"场景，不限学术。

## 页面职责

| 页 | 职责 | 回答的问题 |
|---|---|---|
| index | 总览与入口：各层数量、方向列表、未归类计数 | 我读到哪了、下一步读什么 |
| read | 阅读工作台：摘要卡 + 筛选/搜索 + 草稿标注 | 这份材料说什么、归哪个方向、我怎么判 |
| compare | 方向聚合：每方向的成员卡与判词、未归类桶 | 派系怎么分、空白在哪 |
| map | 溯源图（既有组件） | 设计从哪来、给别人讲清楚 |

## 状态归属（一个事实一个家）

- **方向归属**由 synthesis 卡指向 source 卡的链接推导，不在 source 卡重复记录。
- **判断、阅读状态**的权威版本在卡片 markdown 里；浏览器内的标注只是**草稿**，
  存 localStorage，经 read 页"导出补丁"生成可落盘文本，由 agent 或人写回卡片。
  HTML 永远可以整体重新生成而不丢失任何事实。

## 组件

- 收集器（`build_loom_map` 的 collect）：唯一的 workspace → 数据装配点，map 与 site 共用。
- 站点生成器（`build_loom_site`）：数据 + 页面模板 → 四页；共享主题片段保证同一视觉系统。

## 安全边界

卡片内容视为不可信输入：标题、摘要、frontmatter 值嵌入页面时一律转义或以文本节点呈现，
JSON 内嵌须防 `</script>` 逃逸；生成页零外部请求。对应对抗用例常驻测试。
