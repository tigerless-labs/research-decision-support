# [方法·反面教材] IBIS / QOC — 死于捕获成本的设计论证一族

IBIS（Rittel, 1970，为 wicked problems 设计）· gIBIS/Compendium 工具化 ·
QOC（MacLean et al., 1991, HCI 设计论证）。

核心逻辑：把设计讨论结构化为多名词本体——IBIS 是 issue/position/argument，QOC 是
question/options/criteria。学术上验证了"论证结构化有价值"，工程上整族死亡。尸检结论
（Grudin 悖论）：**承担记录成本的人不是受益的人**——工程师嫌填结构太麻烦，收益归未来
读者，捕获成本杀死整个品类。

边界：作为结构本体仍然有效（QOC 的 question+options 即决定卡骨架）；死的是"人工填表"
这个执行方式。

**与本项目的关系**：两条核心决策的直接依据——只留一个人类实体（多名词=多填表义务）、
记账劳动全部归 agent。整个产品的 why-now 假设：agent 把捕获成本降到零，IBIS 的死因
在今天不再成立（此假设待原型验证，见 decisions 层）。
