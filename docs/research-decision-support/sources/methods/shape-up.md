# [方法] Shape Up — 无 backlog 与裁决桌

Ryan Singer / Basecamp (2019) · 公开出版 · Basecamp 多年实操 · [Bets, Not Backlogs](https://basecamp.com/shapeup/2.1-chapter-07)。

核心逻辑：**backlog 是时间黑洞**——梳理旧想法的劳动挤占当下要事，需要 grooming 的库存
必然腐烂。替代品是 betting table：桌上只有少数几个**成型的**（shaped：风险已收敛、方案
已成形）选项，下注或放掉，nothing to track。对原始想法默认 **soft no**："ideas are
cheap"，真正重要的想法自己会回来——你上次忘掉一个真正好的想法是什么时候？"bet" 一词
刻意选用，承认决策的不确定性。

边界：shaping 由资深成员承担，是前置人力成本；六周节奏等仪式为团队场景设计。

**与本项目的关系**：裁决桌的整体出身——未成熟决定不可见不计数（治"一堆 idea 混乱"）、
只端成型的 2-4 张、没下注的自动放掉、蒸发的合法性。shaping 劳动由 agent 承担（取证+
组矩阵），消除其前置成本。
