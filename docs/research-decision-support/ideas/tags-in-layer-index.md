# 全部 tag 在各层 index 表现：检索走派生索引，不设分类层

tag 的真身只有一处——卡 frontmatter 的 `tags:`。检索不便不靠加结构层解决
（分类层复制事实、必腐烂），靠**派生索引**：每层 index.md 以 tag 为**小标题**
分组，格式 `TAG：一句为什么是一类`，组下列成员行；未分类卡平铺在末尾无标题区。
index 由 agent 随卡变更同回合重生成，永不滞后。改 tag 只改卡，index 自动跟。

先例：Luhmann 的 Zettelkasten 卡片盒旁有 register（关键词索引）作检索入口——
索引是派生物，不是结构层。

[Zettelkasten](../sources/methods/zettelkasten.md) 的 register 惯例；分类层被否
的理由同 [source-tags-self-classify](source-tags-self-classify.md)（固定分类必
错位）。

tag 的唯一性来自 [single-edge-single-tag](single-edge-single-tag.md)；打 tag 的
时机与可选性见 [source-tags-self-classify](source-tags-self-classify.md)；
index 永不滞后同 [output-auto-refresh](output-auto-refresh.md) 的纪律。
