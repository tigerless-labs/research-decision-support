# source 内部 tag 自分类：类型目录是薄壳，主题结构靠 tag 涌现

source 层不预设主题目录树——固定分类是约束，材料一多必然错位。分类靠 **tag 涌现**：
同 tag 即同类，单层且一卡至多一个；
类型容器（论文/仓库/博客/方法/产品）保留但只是薄壳，不承担主题语义。打 tag 发生在
**推导连线步**且**可选**——人主动提出或 agent 自行归类均可（归类是跑腿不是拍板，
人可增删改），无 tag 的卡合法存在、不阻塞引用与装配；改动留 log。
这是"结构从链接长出而非预设"在 source 层的落地，也是少约束理念的直接推论。

[Zettelkasten](../sources/methods/zettelkasten.md) 反目录树、结构后置；本工作区自身
已按类型薄壳运转——主题归属由 synthesis 链接推导而非目录（见
workbench 状态归属（旧工作台设计，已随重设计移除））。

Builds on [one-engine-many-schemas](one-engine-many-schemas.md)（结构从链接长出）与
[read-tag-judge-loop](read-tag-judge-loop.md)（读中打标即分类动作落点）；与
[evidence-grading](evidence-grading.md) 正交——tag 管主题，等级管可信。To be weighed
in [decisions/](index.md)：tag 词表是否收敛、由谁收敛。
