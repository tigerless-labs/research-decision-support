# 计划：loom 多页工作台（read / compare / map / index）

## 背景与目标

现有产物只有一张溯源图（`docs/index.html`），服务"给别人看成果"；缺"给自己用的理解工作台"。
目标：同一个 workspace 生成一个**自包含、零依赖、多页面**静态站，覆盖研究者主循环
"读 → 标 → 判"，简约、通用（不限学术）。

## 产品决定（本次已定）

- **单一输入动作是阅读**：分类、判断在读的过程中完成，不设独立"归类阶段"。
- **多页面**：index（总览）/ read（阅读+分类+判断）/ compare（方向聚合）/ map（溯源图）。
- **HTML 不持有事实**：方向归属由 synthesis 卡 → source 卡的链接推导；浏览器里的标注是
  **草稿**（localStorage），经"导出补丁"回流到 markdown，由 agent/人落盘。
- **通用化文案**：sources 泛指论文/仓库/帖子/任意材料。

## 单元（每单元：docs → tests → code → verify → commit）

1. **设计文档**：`docs/design/loom-site.md`（页面职责、状态归属、安全边界）+ `docs/design/index.md`
   + `docs/index.md`。
2. **收集器扩展**（`tools/build_loom_map.py::collect`）：补 summary（标题后首段）、frontmatter
   透传；红队用例：卡标题含 `</script>`、`<img onerror>` 不得逃逸进生成页。
3. **站点生成器**（`tools/build_loom_site.py` + 模板）：四页生成；read 页筛选/搜索/草稿标注/
   导出补丁；compare 页由 synthesis→source 边聚合，含"未归类"桶。
4. **竞品调研入库**：`docs/research-loom/` 建 sources/products 卡 + synthesis 竞品格局卡 +
   差异化 idea 卡。
5. **demo 重建与收尾**：examples/autoharness 建站到 `docs/`，README / SKILL.md 同步，TODO 清理。

## 验收

- 任一 workspace 一条命令出四页；页面零外部请求；暗/亮主题可用。
- read 页可按类型/方向/状态筛，可留草稿标注并导出为可落盘的补丁文本。
- compare 页每个方向列出成员卡与判词，未被任何方向引用的源卡进"未归类"。
- 注入型卡片内容在四页中均以文本呈现，不执行。
- pytest 全绿；`check_doc_links.py` 通过。

- `tools/check_research_loom.py`（SKILL.md 已引用但不存在）→ 记入 TODO。
