# logs — change ledger (append-only, covering both the ideas and output layers)

Every change (idea creation / **body update** / rename / merge / archival / supersede;
output assembly / edit / sync refresh; and changes to the ledger convention itself) gets
one line auto-appended by the agent; old lines are never edited. Format:
`date · card · action · delta (old → new) · one-line reason` —
**the delta itself must be recorded** (the minimal old → new), not just the action and
reason; the docs stay out of git, so this ledger is the only way back, and one line must
suffice to reconstruct "what changed". Until node-level logs land, this ledger is the only
trace of idea changes. Output sync follows
[output-auto-refresh](ideas/output-auto-refresh.md), and sync actions are ledgered too:
**one line per layer** — a double-write records two lines (one idea, one output), each
with its own old → new, the reason field naming the trigger to chain the causality; before
output exists, only the idea line is written.
