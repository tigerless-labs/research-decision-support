# [Blog] "Notes graveyard" -- the 2026 consensus pain point of the second-brain scene

The same diagnosis across multiple 2026 pieces (Substack/Medium/tool reviews).

Core logic: note systems are "easy in, buried forever" -- strong capture, no retrieval ("funnels
with no drain"); Zettelkasten/PARA push the entire maintenance burden onto the human, and after
two weeks of enthusiasm the archive becomes a grave; "capture without retrieval is hoarding". The
2026 mainstream response is the **AI-maintained vault**: an agent scans the whole vault and
classifies active/stale/superseded/promotable (an Obsidian x Claude Code skill already does this,
tagline "a vault that doesn't review itself is a filing cabinet with extra steps").

**Relevance to this project**: confirms the idea-management pain point is real and the lane is
already noticed; this project's differentiator is not "AI reviews for you" but ideas carrying a
**lifecycle and an evidence obligation** (status enum + mandatory source links +
unused-is-visible). Sources:
[Your Notes Are a Graveyard](https://anshulkumar.substack.com/p/your-notes-are-a-graveyard-heres),
[Claude Code x Obsidian](https://medium.com/@creativeaininja/claude-code-and-obsidian-turn-your-note-taking-app-from-a-graveyard-into-a-living-second-brain-eeed1d1375bf).
