# Source tags self-classify: type directories are thin shells, thematic structure emerges from tags

The source layer presets no thematic directory tree -- fixed classification is a constraint, and
once material piles up it inevitably misfits. Classification relies on **tag emergence**: same
tag, same class; single level and at most one per card. The type containers
(papers / repos / blogs / methods / products) remain but are thin shells only, carrying no
thematic semantics. Tagging happens at the **link-derivation step** and is **optional** --
either the human proposes it or the agent classifies on its own (classifying is running
errands, not adjudicating; the human may add, remove, or change), untagged cards are legal and
block neither references nor assembly; changes leave a log. This is "structure grows from links
rather than being preset" landing on the source layer, and a direct corollary of the
few-constraints tenet.

[Zettelkasten](../sources/methods/zettelkasten.md) is anti-directory-tree, structure comes
later; this workspace itself already runs on thin type shells -- thematic membership is derived
from synthesis links, not directories (see the workbench state ownership (the old workbench
design, removed with the redesign)).

Builds on [one-engine-many-schemas](one-engine-many-schemas.md) (structure grows from links)
and [read-tag-judge-loop](read-tag-judge-loop.md) (tagging while reading is where the
classification action lands); orthogonal to [evidence-grading](evidence-grading.md) -- tags
govern theme, grades govern trust. To be weighed in [decisions/](index.md): whether the tag
vocabulary should converge, and who converges it.
