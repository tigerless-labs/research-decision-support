# SKILL.md and the protocol fuse into one file: structure → workflow → disciplines → quickstart

The human's adjudication (2026-07-14): the skill's operating doc and the protocol contract merge
into a single SKILL.md — two homes kept breeding duplicated rules (the sync invariant said twice
was the proof), and the contract travels with the skill directory either way.

Section order and rationale:

- **Structure first** (layout, layers, card schemas with inline templates, the two structured
  facts, rendering contract): definitions before procedures — every workflow sentence stands on
  these nouns. The engine/methodology boundary survives as a subsection, not a separate file.
- **Workflow second**, in chronological steps (discover → ingest → transcribe → sync on the
  human's command → assemble → project), with teach-in-place attached.
- **Disciplines third**, closing the body: the "never" list, ledger format, validator gate —
  the safety-critical rules take the tail attention position.
- **Quickstart last**: a command runbook appendix.

The standalone `templates/` folder is deleted — necessary skeletons live inline in the schema
section. `references/output-forms/` and `canvas/styles/` stay as separate reference files:
large option sets disclose progressively, the body keeps pointers only.
